
# 2. Protected Access Modifier

```cpp
#include<iostream>
using namespace std;

class InstaStory
{
protected:
    int storyViews = 500;
};

class SponsoredStory : public InstaStory
{
public:
    void showViews()
    {
        cout<<"Views: "<<storyViews;
    }
};

int main()
{
    SponsoredStory s;
    s.showViews();

    return 0;
}
```

# 3. Abstraction (Pure Virtual Function)

```cpp
#include<iostream>
using namespace std;

class Product
{
public:
    virtual void upload() = 0;
};

class Electronics : public Product
{
public:
    void upload()
    {
        cout<<"Electronics Uploaded"<<endl;
    }
};

class Clothing : public Product
{
public:
    void upload()
    {
        cout<<"Clothing Uploaded"<<endl;
    }
};

int main()
{
    Electronics e;
    Clothing c;

    e.upload();
    c.upload();

    return 0;
}
```

# 5. Difference Between Encapsulation and Abstraction

### Encapsulation

Encapsulation means hiding data and accessing it using getter and setter functions.

**Example:** In WhatsApp, phone number is private and accessed through methods.

### Abstraction

Abstraction means showing only important features and hiding internal details.

**Example:** In Instagram, we click the upload button but do not know how the upload process works internally.

### Conclusion

* Encapsulation = Data Hiding
* Abstraction = Hiding Implementation Details
